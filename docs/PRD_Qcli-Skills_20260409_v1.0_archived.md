## 1. 产品概述

### 1.1 产品定位

一款面向个人开发者的 CLI 技能管理工具，支持公共技能、私人技能和知识库的统一管理与多源同步。

### 1.2 核心价值主张

| 价值点 | 说明 |
|--------|------|
| 一站式管理 | 将分散的技能脚本、代码片段、知识笔记统一管理 |
| 多源同步 | 通过 Git 仓库实现公共资源与私人资源的分类同步 |
| 即取即用 | 通过 npx 一键调用，无需全局安装 |
| 安全可控 | 敏感信息检测机制，防止意外泄露 |

## 2. 目标用户画像

| 维度 | 描述 |
|------|------|
| 角色 | 个人开发者 |
| 技术水平 | 熟悉命令行操作，了解 Git 基本用法 |
| 痛点 | 电脑重装后脚本丢失；多台设备间技能不同步；代码片段散落各处 |
| 使用频率 | 每周 3-10 次 |
| 使用环境 | Windows/macOS/Linux 终端 |

### 典型使用场景

1. 在公司电脑上创建一个新技能片段，同步到私人仓库
2. 回家后在个人电脑上拉取最新技能，继续使用
3. 发现一个实用的公共技能，添加到公共仓库并同步
4. 整理项目中的常用代码模板，归档到知识库

### 3.3 同步功能完整定义

#### 增量同步策略

- 系统采用增量同步机制，仅同步自上次同步后发生变更的文件
- 基于文件哈希值（SHA-256）检测变更，避免时间戳不可靠问题
- 同步元数据存储于本地数据库，记录每个文件的最后同步状态

#### 冲突处理策略

**冲突检测条件**：同一文件在本地和远程均被修改，且修改时间均晚于上次同步时间

**冲突处理流程**：
```
检测到冲突 → 暂停同步 → 提示用户选择：
  ├── 保留本地版本（覆盖远程）
  ├── 保留远程版本（覆盖本地）
  ├── 保留双方版本（创建冲突副本）
  └── 取消同步操作
```

**用户交互界面**：
```
⚠️  检测到文件冲突

  文件：docs/README.md
  本地修改时间：2026-04-09 10:30:00
  远程修改时间：2026-04-09 10:25:00

  [1] 保留本地版本
  [2] 保留远程版本
  [3] 保留双方版本
  [4] 跳过此文件

  请选择 (1-4):
```

#### 离线处理机制

| 操作类型 | 离线行为 | 恢复后处理 |
|---------|---------|-----------|
| 本地文件操作 | 正常执行，记录操作日志 | 无需特殊处理 |
| 同步到远程 | 提示"网络不可用，操作已加入待同步队列" | 自动重试队列中的操作 |
| 从远程拉取 | 提示"网络不可用，请稍后重试" | 手动重新执行 |

## 6. 安全相关功能

### 6.1 扫描跳过确认机制

```
⚠️  安全警告

您正在跳过安全扫描。这可能导致敏感信息泄露。

跳过扫描后：
  • 敏感 API 密钥可能被提交
  • 私密配置文件可能被同步
  • 个人凭证可能被推送到远程仓库

确定要跳过吗？

  [y/N] 请输入 'yes' 确认跳过：
```

**确认要求**：
- 必须完整输入 `yes`（不接受 `y` 或其他简写）
- 5 秒内无法输入（防止快速确认）
- 确认后仍记录跳过事件到审计日志

### 6.2 公开仓库推送保护

**推送前强制流程**：

```
检测到目标仓库为公开仓库
────────────────────────────────

🔒 安全扫描中...

扫描结果：
  ✓ 未发现 API 密钥
  ✓ 未发现私钥文件
  ⚠ 发现 2 个潜在敏感文件：
     - config/local.env
     - .secrets/credentials.json

  [查看详情] [继续推送] [取消]

⚠️  警告：您正在推送到公开仓库！

  仓库地址：https://github.com/user/public-repo
  可见范围：所有人可见

  确定要继续推送吗？

  [输入仓库名称确认]：
```

**强制安全措施**：

| 措施 | 说明 |
|-----|-----|
| 强制扫描 | 无法跳过，必须完成安全扫描 |
| 结果展示 | 必须展示扫描结果给用户 |
| 确认输入 | 必须输入仓库名称确认（防止误操作） |
| 审计日志 | 记录推送事件、扫描结果、用户确认 |

## 7. 非功能性需求

### 7.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 启动时间 | ≤ 2 秒 | npx 调用后进入可操作状态 |
| 列表响应 | ≤ 1 秒 | 技能数量 ≤ 500 时 |
| 同步响应 | 取决于网络 | 提供进度反馈，避免用户等待焦虑 |
| 内存占用 | ≤ 100 MB | 正常操作时 |

### 7.2 易用性要求

| 维度 | 要求 |
|------|------|
| 命令设计 | 遵循常见 CLI 规范，命令语义清晰 |
| 帮助文档 | 每个命令都有 `-h/--help` 说明 |
| 错误提示 | 错误信息具体、可操作，包含解决建议 |
| 交互引导 | 交互模式步骤简洁，默认值合理 |
| 输出格式 | 支持表格、JSON 等多种输出格式 |

### 7.3 可靠性要求

| 维度 | 要求 |
|------|------|
| 数据安全 | 不丢失用户数据，同步前做备份检查 |
| 断点续传 | 大文件同步支持中断恢复 |
| 冲突处理 | 同步冲突时提示用户选择处理策略 |

### 7.4 兼容性要求

| 维度 | 要求 |
|------|------|
| 操作系统 | 支持 Windows、macOS、Linux |
| Node.js 版本 | 支持 Node.js 16+ |
| 终端环境 | 兼容常见终端（CMD、PowerShell、Bash、Zsh） |
