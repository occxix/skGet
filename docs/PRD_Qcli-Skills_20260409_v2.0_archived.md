## 1. 产品概述

### 1.1 产品定位

一款面向个人开发者的 CLI 技能管理工具，支持多 AI 编程助手的技能、知识库、Agent 配置的统一管理与多源同步。

### 1.2 核心价值主张

| 价值点 | 说明 |
|--------|------|
| 多环境支持 | 支持 Claude Code、Cursor、通义灵码、OpenAI Codex、CodeBuddy Code 等 AI 编程助手 |
| 一站式管理 | 将分散的技能脚本、代码片段、知识笔记、Agent 配置统一管理 |
| 多源同步 | 通过 Git 仓库实现公共资源与私人资源的分类同步 |
| 即取即用 | 通过 npx 一键调用，无需全局安装 |
| 安全可控 | 敏感信息检测机制，防止意外泄露 |

### 1.3 支持的 AI 环境

| 环境 | 名称 | 配置目录 | 说明 |
|------|------|----------|------|
| `claude` | Claude Code | `~/.claude` | Anthropic Claude Code 助手 |
| `cursor` | Cursor | `~/.cursor` | Cursor AI 代码编辑器 |
| `qwen` | 通义灵码 | `~/.qwen` | 阿里云通义灵码助手 |
| `codex` | OpenAI Codex | `~/.codex` | OpenAI Codex 编程助手 |
| `codebuddy` | CodeBuddy Code | `~/.codebuddy` | 腾讯 CodeBuddy Code 助手 |
| `common` | 通用技能 | 跨环境共享 | 适用于所有环境的通用资源 |

## 2. 目标用户画像

| 维度 | 描述 |
|------|------|
| 角色 | 个人开发者、AI 工具重度用户 |
| 技术水平 | 熟悉命令行操作，了解 Git 基本用法 |
| 痛点 | 多个 AI 工具配置分散；多台设备间技能不同步；代码片段散落各处；Agent 配置重复繁琐 |
| 使用频率 | 每周 3-10 次 |
| 使用环境 | Windows/macOS/Linux 终端 |

### 典型使用场景

1. **场景一：技能快速安装**
   从公共技能库快速安装一个常用技能到 Claude Code 环境

2. **场景二：多环境同步**
   在公司配置好一套 Agent，同步到私人仓库，回家后在个人电脑快速恢复

3. **场景三：批量操作**
   一键将所有技能同步到远程仓库，或从仓库批量下载所有资源

4. **场景四：npx 即用**
   无需初始化配置，直接通过 npx 浏览和安装公共技能

## 4. 功能列表

| P0-01 | 技能添加 | 支持添加单文件或文件夹形式的技能到指定环境 | 成功添加后可列出该技能 |

## 5. 用户故事

### Story 1：从公共技能库快速安装
> **作为** 开发者
> **我希望** 通过 `qskills install code-review --env claude` 直接从公共技能库安装技能
> **以便** 快速获得常用功能，无需手动创建文件

### Story 2：批量同步多个环境
> **作为** 在多台电脑工作的开发者
> **我希望** 一键将所有技能同步到私人仓库
> **以便** 在其他设备上快速恢复工作环境

### Story 3：Agent 快速部署
> **作为** AI 工具重度用户
> **我希望** 将配置好的 Agent 一键部署到多个 AI 环境
> **以便** 避免在每个环境中重复配置

### Story 4：离线查看公共技能
> **作为** 开发者
> **我希望** 在无网络时仍能查看内置公共技能列表
> **以便** 了解有哪些可用资源

### Story 5：版本冲突解决
> **作为** 团队协作者
> **我希望** 在同步时检测到版本冲突后能清晰选择保留哪个版本
> **以便** 避免意外覆盖重要修改

### Story 6：npx 即用模式
> **作为** 新用户
> **我希望** 无需初始化配置就能通过 npx 浏览公共技能
> **以便** 快速了解工具价值

## 7. 命令设计

### 7.1 核心命令

```bash
# 资源安装
qskills install <name>           # 从公共库安装
qskills install <name> --env claude --from repo

# 资源下载
qskills download [--env <env>] [--source <repo>]
qskills download --all           # 下载全部

# 资源同步
qskills sync [--env <env>] [--source <repo>]
qskills sync --all               # 同步全部

# 资源管理
qskills skill add <path> --env <env>
qskills skill list [--env <env>]
qskills skill remove <name> --env <env>

qskills knowledge add <path> --env <env>
qskills knowledge list [--env <env>]
qskills knowledge remove <id> --env <env>

qskills agent add <path> --env <env>
qskills agent list [--env <env>]
qskills agent deploy <name> --env <env>
qskills agent remove <name> --env <env>

# 环境管理
qskills env list                 # 列出所有环境
qskills env copy <name> --from <env> --to <env>

# 配置
qskills config init
qskills config set <key> <value>
qskills config list
```

### 7.2 npx 模式

```bash
# 无配置模式 - 只显示内置公共技能
npx qskills list

# 有配置模式 - 显示仓库技能 + 内置技能
npx qskills list --repo <url> --token <token>

# 快速安装
npx qskills install <name> --env claude
```

## 8. 非功能性需求

### 8.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 启动时间 | ≤ 2 秒 | npx 调用后进入可操作状态 |
| 列表响应 | ≤ 1 秒 | 资源数量 ≤ 500 时 |
| 同步响应 | 取决于网络 | 提供进度反馈 |
| 内存占用 | ≤ 100 MB | 正常操作时 |

### 8.2 易用性要求

| 维度 | 要求 |
|------|------|
| 命令设计 | 遵循常见 CLI 规范，命令语义清晰 |
| 帮助文档 | 每个命令都有 `-h/--help` 说明 |
| 错误提示 | 错误信息具体、可操作，包含解决建议 |
| 交互引导 | 交互模式步骤简洁，默认值合理 |
| 输出格式 | 支持表格、JSON 等多种输出格式 |

### 8.3 可靠性要求

| 维度 | 要求 |
|------|------|
| 数据安全 | 不丢失用户数据，同步前做备份检查 |
| 冲突处理 | 同步冲突时提示用户选择处理策略 |
| 离线支持 | 离线时可查看本地资源，操作队列延迟同步 |

### 8.4 兼容性要求

| 维度 | 要求 |
|------|------|
| 操作系统 | 支持 Windows、macOS、Linux |
| Node.js 版本 | 支持 Node.js 16+ |
| 终端环境 | 兼容 CMD、PowerShell、Bash、Zsh |

## 10. 术语定义

| 术语 | 定义 |
|------|------|
| Skill（技能） | 可复用的脚本、工具或代码片段，以文件或文件夹形式存在 |
| Knowledge（知识库） | 文档笔记和代码模板的集合 |
| Agent | AI Agent 配置包，包含 system prompt、tools、context 等完整配置 |
| 环境 | AI 编程助手的配置目录，如 ~/.claude、~/.cursor 等 |
| 公共仓库 | 存储公共资源的 Git 仓库，可公开分享 |
| 私人仓库 | 存储私人资源的 Git 仓库，私有访问 |
| 内置公共库 | 工具自带的默认技能/知识/Agent 集合 |
